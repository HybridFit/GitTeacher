# The Git Gym — Interactive Git Learning Repository

> **For Claude:** Use envoy:executing-plans to implement this spec task-by-task.

## Overview

A repository that teaches Git through hands-on assignments themed around a fictional gym called "The Git Gym." Each assignment presents a realistic scenario (coaches editing schedules, creating workout plans) that requires specific Git skills to resolve. Pre-built branches provide conflict scenarios that are hard to self-create. A `solutions/` folder offers hints after attempting.

## Architecture

### Repository Structure

```
README.md                  — Welcome, setup instructions, how to use

  01-branching.md          — "New Training Program" — create and merge a feature branch
  02-merge-conflict.md     — "Schedule Clash" — resolve a merge conflict
  03-rebase.md             — "Clean Up Your Plan" — interactive rebase messy commits
  04-collaboration.md      — "Shift Handover" — push, pull, handle remote conflicts
  05-worktrees.md          — "Multitasking Coach" — use worktrees for parallel work
solutions/
  01-solution.md
  02-solution.md
  03-solution.md
  04-solution.md
  05-solution.md
```

### Content Files (on main branch)

- `workout-plan.txt` — current gym workout plan (exercises, sets, reps)
- `schedule.txt` — weekly class timetable
- `nutrition-guide.txt` — basic meal plan for clients
- `clients.txt` — client roster with names and goals

### Pre-built Branches

| Branch | Purpose | State |
|--------|---------|-------|
| `feature/schedule-redesign` | Merge conflict exercise | Edited `schedule.txt` differently from main |
| `feature/messy-plan` | Rebase exercise | 5+ sloppy commits editing `workout-plan.txt` |
| `friend/coach-updates` | Collaboration exercise | Simulates a colleague's pushed changes |

### Assignment Design

Each assignment follows this structure:
1. **Scenario** — a story explaining why they need to do this
2. **Your Task** — what they need to accomplish
3. **Useful Commands** — a reference list (not a walkthrough)
4. **Check Yourself** — how to verify success (both in terminal and in Fork)

### Target Audience

Someone who uses Fork as a Git GUI, understands commits and branches visually, but needs command-line skills and deeper understanding of rebase, conflict resolution, and worktrees.

**Files:**

**Steps:**
1. Initialize git repo: `git init`
2. Create `workout-plan.txt` with a full-body workout plan (8-10 exercises with sets/reps)
3. Create `schedule.txt` with a weekly class timetable (Monday–Saturday, 3-4 classes per day)
4. Create `nutrition-guide.txt` with a basic meal plan (breakfast, lunch, dinner, snacks)
5. Create `clients.txt` with 6-8 fictional client names and their fitness goals
6. Create `README.md` with welcome text, setup instructions, and assignment overview
7. Commit all files: `git commit -m "feat: initial Git Gym content and README"`

#### Task 2: Create pre-built branch `feature/schedule-redesign`

**Files:**
- Modify: `schedule.txt`

**Steps:**
1. From main, create branch: `git checkout -b feature/schedule-redesign`
2. Edit `schedule.txt` — change Monday and Wednesday class times, add a new Saturday class, remove one Tuesday class
3. Commit: `git commit -m "feat: redesign weekly schedule with new time slots"`
4. Return to main: `git checkout main`
5. Edit `schedule.txt` on main differently — change Monday classes to different times, modify Tuesday
6. Commit on main: `git commit -m "feat: update schedule based on member feedback"`

This ensures merging `feature/schedule-redesign` into main will produce conflicts.

#### Task 3: Create pre-built branch `feature/messy-plan`

**Files:**
- Modify: `workout-plan.txt`

**Steps:**
1. From main, create branch: `git checkout -b feature/messy-plan`
2. Make 5-6 commits with messy messages:
   - "wip" — add one exercise
   - "more stuff" — add another exercise
   - "fix typo" — fix a typo in the previous addition
   - "oops forgot reps" — add missing rep counts
   - "WIP saving before lunch" — reorder two exercises
   - "final maybe" — add a warmup section
3. Return to main: `git checkout main`

#### Task 4: Create pre-built branch `friend/coach-updates`

**Files:**
- Modify: `clients.txt`
- Modify: `nutrition-guide.txt`

**Steps:**
1. From main, create branch: `git checkout -b friend/coach-updates`
2. Edit `clients.txt` — add 2 new clients, update one existing client's goal
3. Commit: `git commit -m "feat: add new January clients"`
4. Edit `nutrition-guide.txt` — change the snack section, add a vegetarian option
5. Commit: `git commit -m "feat: update nutrition guide with vegetarian options"`
6. Return to main: `git checkout main`
